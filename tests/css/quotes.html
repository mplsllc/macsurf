MacSurf — fixes140 (quotes + open/close-quote)

fixes140 wires the four quote content items (open-quote, close-quote, no-open-quote, no-close-quote) and the quotes property. Depth tracks across the document so nested <q> produces the secondary quote pair from the list.

PROBE Q1 — UA default quotes via <q>

The captain said land ho from the crow's nest.

EXPECT: land ho wrapped in curly double quotes from the UA stylesheet. No author CSS needed -- fixes140c adds the q rule and ::before/::after defaults inline in the macos9 resource CSS.

PROBE Q2 — nested <q> uses the second pair

She replied he said land ho twice.

EXPECT: outer ... uses curly double; inner ... uses curly single. Depth increments through open-quote, decrements through close-quote, so nesting picks the second pair.

PROBE Q3 — author-defined ASCII quotes

Outer level one level two back out again.

EXPECT: outer wrapped in << >>, inner wrapped in < >. Same depth behaviour as Q2 but with author-supplied strings.

PROBE Q4 — no-open-quote / no-close-quote

silent: emits nothing on either side but still bumps quote depth.

EXPECT: text reads exactly as authored, plus the literal trailing string " (no close)" from the ::after content. No quote characters are emitted. fixes140a still updates depth so later siblings see the right nesting level.

PROBE Q5 — depth carries across paragraphs

First alpha

Second beta gamma

EXPECT: alpha and beta wrapped in curly double (depth 0), gamma in curly single (depth 1). Depth resets only via balanced close-quote pairs, not by paragraph boundaries.
