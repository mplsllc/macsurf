# `transform` — single-feature plan (post-fixes70)

Scope: implement enough of CSS3 `transform` that rotated / scaled /
translated boxes render natively on Mac OS 9. Visually impressive,
nobody expects 2D-transformed CSS on a beige G4.

## Goal — what works at the end of this round

Author CSS like:

```css
.badge   { transform: rotate(-12deg); }
.tooltip { transform: translate(20px, -10px) scale(1.1); }
.spin90  { transform: rotate(90deg); }
.skew    { transform: matrix(0.9, 0, 0, 0.9, 0, 0); }
```

renders with the boxes (backgrounds, borders, box-shadows) honouring
the transform. Text rotates cleanly at 0° / 90° / 180° / 270°; at
arbitrary angles text renders upright at the transformed origin
(V1 limitation, documented).

Test page: a new `transform.html` with rotated badges, scaled hover
states, translated tooltips, and a 90°-rotated heading.

## Non-goals for this round

- `perspective()` / 3D transforms — `rotate3d`, `rotateX`, `rotateY`,
  `rotateZ`, `translateZ`, `matrix3d`. Out of scope; needs perspective
  projection and z-sorting we won't ship in one round.
- Free-angle (non-90°) text rotation. Requires offscreen GWorld
  rendering and per-pixel rotation. Queued as a V2 follow-up.
- `transform-origin` non-default. Default origin is element centre
  (50% 50%); ship that. Custom origins can come in V2.
- `transform-style`, `backface-visibility`, `will-change`. Cosmetic
  hints, not core behaviour.
- `transition` / `animation` interpolating transforms. Stack on this
  in a separate round once `transform` itself is stable.

## CSS parse — what we need to recognise

`transform: none | <function>+`

Function vocabulary for V1:

| Function | Args | Stored as |
|---|---|---|
| `rotate(<angle>)` | one angle in deg/rad/grad/turn | matrix |
| `translate(<x>, [<y>])` | one or two lengths/percentages | matrix |
| `translateX(<x>)` | one length | matrix |
| `translateY(<y>)` | one length | matrix |
| `scale(<sx>, [<sy>])` | unitless multiplier(s) | matrix |
| `scaleX(<sx>)`, `scaleY(<sy>)` | one multiplier | matrix |
| `skew(<ax>, [<ay>])` | one or two angles | matrix |
| `skewX(<ax>)`, `skewY(<ay>)` | one angle | matrix |
| `matrix(a, b, c, d, e, f)` | six numbers | matrix directly |

Multi-function values compose right-to-left:
`transform: A B C` = mat(A) × mat(B) × mat(C) × point.

## Storage

A 2D affine matrix is six numbers:

```
[ a  c  e ]   [ x ]   [ a*x + c*y + e ]
[ b  d  f ] × [ y ] = [ b*x + d*y + f ]
[ 0  0  1 ]   [ 1 ]   [       1       ]
```

Computed style holds a pointer (or arena handle) to a 6-element
fixed-point matrix. `NULL` / sentinel = identity = no transform.
Reuses the existing libcss arena allocator the way custom-properties
do — no new memory subsystem.

Why pointer, not bit-packed inline: 6 × 32-bit = 24 bytes of payload
won't fit a free bit slot in `autogenerated_computed.h`. The
text-shadow / gradient properties bit-packed because each fit in 32
bits; a transform matrix doesn't.

`plot_style_t` gains:
```c
struct plot_transform {
    css_fixed a, b, c, d, e, f;   /* matrix entries */
    int origin_x, origin_y;       /* element-relative pivot, px */
};
const struct plot_transform *transform;  /* NULL = identity */
```

Initial: NULL pointer. Cascade resolves transform at element layout
time using element's box centre as origin.

## Plot pipeline

`macos9_plotters` callbacks (rectangle / line / disc / arc / path /
bitmap / text / clip) gain transform-aware versions that:

1. Read `ctx->transform` if set, else short-circuit to current code.
2. Convert the plot's coordinates into a small quad / path.
3. Apply `transform_apply_to_point(matrix, &x, &y)` per corner.
4. Hand the transformed coordinates to QuickDraw via:
   - **Rectangles & fills** — QuickDraw `FillPoly` / `FramePoly` of
     the transformed quad. Or build an RgnHandle and `FillRgn`.
   - **Borders** — four transformed line segments per side. Drawn as
     transformed `Line()` calls.
   - **Lines** — transform both endpoints, `MoveTo` / `LineTo`.
   - **Bitmaps** — `CopyBits` with rotated `srcRect` is NOT a thing
     in QuickDraw; would need GWorld offscreen + manual pixel rotate.
     V1 fallback: draw bitmap at the transformed-origin, no rotation
     of the bitmap itself (just translation/scale-friendly).
   - **Text** — see below.

## Text under transform

Three paths:

1. **Identity / pure translate / pure scale** — apply origin offset
   to `MoveTo`, render text normally. Free.
2. **Exact 90° / 180° / 270° rotation** — hand-pivot.
   - 180°: `MoveTo` to bottom-right of the transformed box, draw text
     with `TextMode(srcXor)` after a QuickDraw flip — or render
     reverse-order chars with `CharWidth` offsets.
   - 90° / 270°: draw each glyph at a vertically offset position,
     advancing in the perpendicular direction. One `DrawText`-per-glyph
     loop, similar to the existing letter-spacing path in `plot_text`.
3. **Arbitrary angle** — V1 punts. Render text upright at the
     transformed origin, log a one-off warning. V2 will GWorld+rotate.

## Origin / pivot

CSS default is `transform-origin: 50% 50%` — element centre. To apply
a rotation at the centre instead of the top-left corner, the box's
plot loop must:

1. Compute `(cx, cy)` = box centre at the box's natural position.
2. Before applying matrix to each point, subtract `(cx, cy)`.
3. After matrix, add `(cx, cy)` back.

This is composed into the matrix once per box:
```
M' = T(cx,cy) × M × T(-cx,-cy)
```

`plot_transform.origin_x/y` carries this for the plot pass so each
call site doesn't recompute.

## Risk register

| Risk | Mitigation |
|---|---|
| Matrix in computed-style storage doesn't fit | Use pointer field, not bit-packed; allocate from libcss arena |
| QuickDraw `FillPoly` on rotated quad slow at every redraw | Cache transformed RgnHandle per element, invalidate on reformat |
| Text rotation 90° looks like garbage on AA fonts | Force `SetAntiAliasedTextEnabled(false)` for rotated text runs only |
| Fixed-point matrix overflow on large coordinates | Clamp box coords before transform; warn if box > 16k px |
| Author CSS sets a transform on `body` or `html` | Document and refuse — `body` transforms break scroll math |
