# CSS / HTML Standards Coverage Audit — 2026-05-22

Snapshot of MacSurf's CSS 2.1, CSS3-module, and HTML5 conformance just after
fixes174 (Apple loads without crashing). Drives the next 10+ rounds of
spec-compliance work.

The user directive that produced this doc: **"We should be building to CSS
standards. The websites are there as a guide, but we should be building
[CSS/HTML] as a whole, not just targeting websites."**

For each entry the state is one of:

- **PARSED** — libcss has a parser (`p_*.c` or `autogenerated_*.c`).
- **CASCADED** — value reaches `css_computed_style` via dispatch.
- **CONSUMED** — layout reads it via `css_computed_<prop>()`.
- **PAINTED** — redraw reacts to it.
- **MISSING** — not present at any layer.

A property is "fully wired" only when all four are true. The interesting
gaps are properties that are **PARSED + CASCADED but not CONSUMED**: libcss
silently drops the declaration into the cascade and nothing reads it. From
the author's perspective the property is dead.


## CSS 2.1 conformance gaps

These are required for baseline CSS 2.1 conformance.

| Property | State | Notes |
| --- | --- | --- |
| `caption-side` | PARSED + CASCADED, NOT CONSUMED | `<caption>` always renders inline above the table; CSS spec says top/bottom positioned. Layout would need to detect the value and reposition the caption box. |
| `table-layout: fixed` | PARSED + CASCADED, NOT CONSUMED | Auto-layout is the only path. Authors can't pre-size table columns. Documented in CLAUDE.md as deferred (1064 lines in table.c). |
| `orphans`, `widows` | PARSED + CASCADED, NOT CONSUMED | Print-only typography. Lowest priority for an interactive browser. |
| `page-break-*` | PARSED + CASCADED, NOT CONSUMED | Print-only. Same priority. |
| `unicode-bidi` | PARSED + CASCADED, NOT CONSUMED | Right-to-left text layout. Real impact only on RTL-language pages. |
| `empty-cells` | shipped at fixes139a | Listed only for completeness — already works. |
| `word-spacing` | shipped at fixes139b | Same. |

CSS 2.1 selectors and at-rules (`:link`/`:visited`/`:hover`/`:active`,
`@import`, `@media`, `@font-face`, attribute selectors, descendant
combinators) are fully wired via libcss's stock implementation.


## CSS3 module gaps

### Selectors Level 3

`:nth-child()`, `:nth-of-type()`, `:not()`, `:empty`, `:first-of-type`,
`:last-of-type`, `:only-child`, `:target` — all wired through libcss.

`:focus`, `:focus-visible`, `:focus-within` — depend on whether MacSurf
tracks a focused element. URL field has `url_field_active`; box-level focus
is not tracked. Probably **NOT CONSUMED** even if cascaded.

### Backgrounds & Borders Level 3

| Property | State | Notes |
| --- | --- | --- |
| `background-size` | **MISSING ENTIRELY** | Not in propstrings.h, no parser, no consumer. Modern responsive heroes use `cover` / `contain`. Adding it requires a new libcss property entry, which CLAUDE.md flags as a mid-struct-insert crash risk (must go at end of `_i`). |
| `background-origin`, `background-clip` | MISSING ENTIRELY | Modest impact; default `padding-box` matches most authors. |
| `background-repeat-x`, `background-repeat-y` | MISSING (longhands) | The `repeat`/`no-repeat`/`repeat-x`/`repeat-y` shorthand path works (fixes138). |
| `border-image-*` | MISSING ENTIRELY | Decorative; degrades to plain border. |
| `border-radius` | wired at fixes172 | — |
| `box-shadow` | wired at fixes175 | — |

### Text Level 3

| Property | State | Notes |
| --- | --- | --- |
| `text-shadow` | **VENDOR ONLY** — `-macsurf-text-shadow` is wired (fixes50). Standard `text-shadow` MISSING from libcss. | Highly common in real CSS. Bridgeable via cssh_css preprocessor (same pattern as fixes115 grid-template-columns). |
| `word-break: break-all` / `overflow-wrap: break-word` | PARSED + CASCADED, partial consumption | MacSurf's default already character-breaks long unbreakable runs, so visual difference is small. Documented in CLAUDE.md fixes136. |
| `hyphens`, `text-justify`, `text-align-last` | MISSING ENTIRELY | Low impact for an OS 9 browser. |
| `tab-size` | PARSED, NOT CONSUMED | Code blocks render tabs at the QuickDraw default. |

### Transforms / Transitions / Animations

| Property | State | Notes |
| --- | --- | --- |
| `transform` | **VENDOR ONLY** — `-macsurf-transform` is wired (fixes73). Standard `transform` MISSING. | fixes141 attempted to alias at propstrings level and HUNG the browser pre-reformat (see CLAUDE.md). Future bridge must use cssh_css preprocessor route, not parser-level aliasing. |
| `perspective`, `transform-style`, `backface-visibility` | MISSING ENTIRELY | CSS 3D is out of scope for a 2D QuickDraw backend. |
| `transition`, `animation` | **VENDOR ONLY** — `-macsurf-animation-opacity` / `-macsurf-animation-rotate`. Standard `transition`/`animation` MISSING. | Bridging means more than name rewriting — animation timing functions and keyframes need parser entries. Deferred to v0.4.5. |

### Fonts Level 3

| Property | State | Notes |
| --- | --- | --- |
| `font-feature-settings` | MISSING ENTIRELY | QuickDraw text has no OpenType feature dispatch. |
| `font-variant-*` longhands | MISSING | `font-variant: small-caps` (the legacy shorthand) is parsed. |
| `font-display` | MISSING ENTIRELY | Web font swap behavior. |
| `font-stretch` | PARSED, NOT CONSUMED | QuickDraw has no condensed/expanded faces. |

| `filter`, `clip-path`, `mask` | MISSING ENTIRELY | Decorative; pages render without visual effect. |
| `appearance`, `user-select`, `pointer-events` | MISSING ENTIRELY | UI-level; mostly affects forms and selection behavior. |

## HTML5 element + form gaps

### Semantic elements

| Element | Default | Notes |
| --- | --- | --- |
| `<article>`, `<section>`, `<nav>`, `<aside>`, `<header>`, `<footer>`, `<main>` | `display: block` from UA stylesheet | Wired in macos9_fetcher_stubs.c. Renders correctly. |
| `<figure>`, `<figcaption>`, `<hgroup>` | `display: block` | Same. |
| `<details>`, `<summary>` | `display: block`, **no toggle behavior** | The disclosure widget doesn't open/close on click. Requires a mouse-event hook + DOM mutation of the `open` attribute. Modest scope. |
| `<dialog>` | `display: block`, **no modal behavior** | Modal stacking / backdrop / `showModal()` semantics not implemented. |
| `<template>` | Doesn't suppress rendering | Content is rendered instead of being inert. |

### Media & embedding

| Element | State |
| --- | --- |
| `<img>`, `<picture>` | `<img>` wired (fixes78k); `<picture>` / `srcset` NOT — first source always used. |
| `<video>`, `<audio>` | NOT SUPPORTED. No codec. |
| `<iframe>` | NOT SUPPORTED. |
| `<canvas>` | NOT SUPPORTED. |
| `<svg>` (inline) | NOT SUPPORTED. SVG embedded via `<img src=*.svg>` doesn't paint either (no SVG decoder). |
| `<embed>`, `<object>` | LIMITED. `object.c` exists; minimal feature coverage. |

### Form elements

| Input type | State |
| --- | --- |
| `text`, `submit`, `button`, `checkbox`, `radio`, `password`, `hidden` | WIRED via form.c. |
| `email`, `tel`, `url`, `search`, `number`, `range`, `date`, `time`, `color` | NOT SUPPORTED — fall through as plain text inputs. No client-side validation. |
| `file`, `datetime-local`, `month`, `week` | NOT SUPPORTED. |
| `<input list=>` + `<datalist>` | NOT SUPPORTED. |
| `<input pattern>`, `min/max/step` | NOT SUPPORTED. |
| `<meter>`, `<progress>`, `<output>` | NOT SUPPORTED. |
| Form-level constraint validation (`required`, `:invalid`, `:valid`) | NOT SUPPORTED. |


## Prioritisation for the next 10 rounds

The "build CSS/HTML as a whole" directive means picking gaps by **spec
weight × author frequency × implementation cost**, not by which site they
unblock.

### Round-class A — quick standards bridges (cheap, high real-world usage)

- **`text-shadow` standard alias.** Vendor parser exists. Bridge via
  cssh_css preprocessor, same pattern as fixes115 grid-template-columns.
  Single file. **This is fixes175.**
- **`<details>` / `<summary>` toggle behavior.** Mouse handler in
  `interaction.c` plus a DOM-mutation helper to flip the `open`
  attribute. Two-file fix.
- **`background-clip` / `background-origin` defaults.** Autogenerated
  parsers can produce CASCADED values; redraw can switch on them when
  computing the bg image clip rect.

### Round-class B — standalone consumers for already-cascaded properties

These have no libcss work — the parser & cascade already populate the
computed style. Need only a redraw or layout consumer.

- **`caption-side`** — reposition caption box.
- **`column-count` 1-pass simple consumer** — for the simple case where the
  layout would otherwise be single-column anyway, splitting into N
  vertical bands. Skip column-rule-* for V1.
- **`writing-mode` (logical-vs-physical mapping)** — degraded to LTR. Real
  vertical / RTL needs a global layout refactor; skip until real demand.

### Round-class C — new libcss property entries (more risk)

These all hit the mid-struct-insert / dispatch-dual-sync gotchas
(CLAUDE.md). Each one is a multi-file ship with a real risk of crashing
the cascade. Save until the round-class A and B work is exhausted.

- `background-size`.
- `border-image-*`.
- `filter`, `clip-path`, `mask` (likely deferred indefinitely — no
  QuickDraw equivalent of any of these).

### Round-class D — new layout modes / element behaviors

- `<picture>` + `srcset` resolution.
- `<video poster>` (render the poster image, ignore the video itself).
- `<iframe>` nested document layout.
- HTML5 form input types (date, time, range, etc.) — each needs a new
  Carbon control.

### Round-class E — animation pipeline (largest)

- Standard `transition` and `animation`. Vendor implementations exist but
  the standard syntax has timing functions, keyframes, multi-property
  declarations, etc. Whole new parser scope; deferred to v0.4.5.


## What we are NOT chasing

- Per-pixel image alpha blending beyond the existing PNG 1-bit mask path.
- WebGL, WebRTC, WebAssembly, ServiceWorker, IndexedDB — out of scope for
  a Carbon CFM browser by orders of magnitude.
- ECMAScript 6+ features in JS — Duktape is ES5; modern frameworks belong
  on the proxy render-and-flatten path, not in-browser.

The motivating directive again: **build the spec, use sites as evidence
of gaps**. This audit is the map for that build.
